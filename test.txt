# Aegis RAG Pipeline - Testing Guide

This guide will help you set up and test the complete RAG pipeline.

## Prerequisites

1. Python 3.10+
2. Gemini API Key (get from https://aistudio.google.com/app/apikey)

---

## Step 1: Install Dependencies

Create a virtual environment and install required packages:

```bash
# Create virtual environment
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (Linux/Mac)
source venv/bin/activate

# Install dependencies
pip install fastapi uvicorn python-multipart pydantic-settings
pip install PyPDF2
pip install google-genai
pip install numpy
pip install sentence-transformers  # Optional - for vector search
```

---

## Step 2: Configure Environment

1. Copy the example environment file:
```bash
copy .env.example .env
```

2. Edit `.env` and add your Gemini API key:
```
GEMINI_API_KEY=your_actual_api_key_here
```

---

## Step 3: Start the API Server

```bash
# Run the server
python -m uvicorn src.aegis.api.main:app --reload --port 8000
```

Or run directly:
```bash
python src/aegis/api/main.py
```

The API will start at http://localhost:8000

---

## Step 4: Test the Pipeline

### Option A: Using curl

#### 1. Check Health
```bash
curl http://localhost:8000/health
```

Response:
```json
{
  "status": "healthy",
  "version": "2.0.0",
  "documents": 0,
  "chunks": 0,
  "llm": "gemini-2.0-flash"
}
```

#### 2. Upload a Document

Create a test file first (e.g., `test_doc.txt`):
```
The quick brown fox jumps over the lazy dog.
Machine learning is a subset of artificial intelligence.
Deep learning uses neural networks with multiple layers.
Natural language processing deals with text data.
Python is a popular programming language for data science.
```

Upload it:
```bash
curl -X POST http://localhost:8000/upload \
  -F "file=@test_doc.txt"
```

Response:
```json
{
  "document_id": "abc-123",
  "filename": "test_doc.txt",
  "status": "uploaded",
  "chunks_count": 3,
  "message": "Document uploaded and indexed with 3 chunks"
}
```

#### 3. Query the Document

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What is machine learning?",
    "top_k": 3,
    "fusion": "rrf",
    "use_llm": true
  }'
```

Response:
```json
{
  "answer": "Machine learning is a subset of artificial intelligence...",
  "sources": [
    {
      "chunk_id": "doc_1_0",
      "text": "Machine learning is a subset of artificial intelligence.",
      "score": 0.85,
      "rank": 1,
      "document_id": "abc-123"
    }
  ],
  "query": "What is machine learning?"
}
```

### Option B: Using the Web UI

2. Use the interactive Swagger UI to:
   - Upload documents via POST /upload
   - Query via POST /query

## API Endpoints Summary

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | / | API info |
| GET | /health | Health check |
| POST | /upload | Upload document (multipart form) |
| GET | /documents | List all documents |
| GET | /documents/{id} | Get document details |
| DELETE | /documents/{id} | Delete document |
| POST | /query | Query RAG system |
| GET | /search | Simple search (GET) |

---

## Query Options

```python
{
    "query": "Your question here",
    "top_k": 5,           # Number of results to return
    "fusion": "rrf",      # "rrf" or "weighted"
    "use_llm": true       # Use Gemini to generate answer
}
```

## Testing with PDFs

Upload a PDF instead of text:
```bash
curl -X POST http://localhost:8000/upload \
  -F "file=@my_document.pdf" \
  -F "chunking_strategy=pdf"
```

---

## Troubleshooting

### "No relevant documents found"
- Make sure you've uploaded a document first
- Check that the document was processed (chunks_count > 0)

### "Error generating answer"
- Verify GEMINI_API_KEY is set in .env
- Check the API key is valid

### Import errors
- Install all dependencies: `pip install -r requirements.txt`

## Configuration

Edit `.env` to customize:

```env
# Chunking
DEFAULT_CHUNK_SIZE=512
DEFAULT_CHUNK_OVERLAP=50

# Search
DEFAULT_TOP_K=5
BM25_WEIGHT=0.5
RRF_K=60

# LLM
LLM_MODEL=gemini-2.0-flash
LLM_MAX_TOKENS=2048
LLM_TEMPERATURE=0.7
```

http://127.0.0.1:8000/docs
